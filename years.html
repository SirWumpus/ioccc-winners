About the 4th IOCCC (1987)
Download all IOCCC winning entries of 1987
  • 1987/biggar - Best abuse of the rules - program defined on the command line
  • 1987/heckbert - Best obfuscator of programs - obfuscated fold program
  • 1987/hines - Worst style - counts goto’s with all ids goto anagrams with all goto flow
  • 1987/korn - Best one liner - one-liner that prints a message on a Un*x system
  • 1987/lievaart - Grand Prize - very good Othello player
  • 1987/wall - Most useful obfuscation - roman numeral -> decimal and vice versa conversion
  • 1987/westley - Best layout - individually palindromic lines prints a palindrome

1986 - The 3rd IOCCC

About the 3rd IOCCC (1986)
Download all IOCCC winning entries of 1986
  • 1986/applin - Most adaptable program - hello world works under cc and /bin/sh and f77
  • 1986/august - Best complex task done in a complex way - struct-heavy e-computation with memory allocation with sbrk
  • 1986/bright - Most useful obfuscation - hex dump with cpp compressed that uses lots of << for constants
  • 1986/hague - Worst abuse of the C preprocessor - morse code filter written with ’DIT DAH _DIT DIT_DAH’
  • 1986/holloway - Best simple task performed in a complex way - prints hello world! via a recursive main
  • 1986/marshall - Best layout - train engine-shaped program prints choo choo
  • 1986/pawka - Most illegible code - prints ‘obfuscated?’ big using xyyyx idents
  • 1986/stein - Best one liner - prints HHMMSS in UTC time
  • 1986/wall - Grand Prize - Most well rounded in confusion - Dvorak keyboard emulator

1985 - The 2nd IOCCC

About the 2nd IOCCC (1985)
Download all IOCCC winning entries of 1985
  • 1985/applin - Best one liner - one-liner to print hello world with execlp
  • 1985/august - Most obscure program - cpp obfuscated and compressed recursive primes in binary
  • 1985/lycklama - Strangest appearing program - heavily cpped with o_oo identifiers that scrolls text right
  • 1985/shapiro - Grand Prize - Most well rounded in confusion - two #defines one 7-liner that results in a maze
  • 1985/sicherman - Worst abuse of the C preprocessor - rot13 w/strange #defines that create comment headers etc.
